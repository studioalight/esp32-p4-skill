# Issue: `iterate` Command Uploads Wrong (Cached) Binary

**Status:** Open  
**Priority:** High (breaks core workflow)  
**Labels:** bug, workflow, flash

---

## Problem Statement

The `esp32-p4 iterate --project . --monitor` command builds the correct versioned binary but uploads a **stale cached version**, leading to confusing "changes don't appear on device" scenarios.

### Observed Behavior

Running `iterate` produces:
```
✓ Build successful!
  esp_brookesia_demo-bfed6de-dirty.bin: 5,465,584 bytes (versioned)

Uploading from: .../esp_brookesia_phone_iterating
  Uploading esp_brookesia_demo-a66b95a-dirty.bin...   ← OLD!
  Uploading esp_brookesia_demo-bfed6de-dirty.bin...   ← NEW (missing!)
```

The device ends up running the old binary despite the new one being built.

---

## Root Cause

The `upload` step in `iterate` relies on the `flash_args` manifest generated by ESP-IDF. This manifest references filenames recorded at a previous point in time.

**Sequence:**
1. Build creates **new** file: `project-<new_commit>.bin`
2. `flash_args` manifest still references **old** file: `project-<old_commit>.bin`
3. Upload sends old file to bridge
4. Flash flashes stale binary

**Gap:** Versioned binaries (`project-<hash>-dirty.bin`) vs. static names (`app.bin`)

---

## Workaround (Current)

Manual file specification bypasses manifest discovery:

```bash
# After build
esp32-p4 upload --file build/project-<commit>-dirty.bin
esp32-p4 flash --file project-<commit>-dirty.bin --addr 0x10000
```

**Note:** This breaks the seamless `iterate` workflow and requires manual commit hash lookup.

---

## Proposed Fixes

### Option A: Update flash_args Post-Build
Update the manifest after each build to reference the actual versioned binary:

```python
# After build completes
git_commit = get_current_commit()
update_flash_args_manifest('app.bin', f'project-{git_commit}.bin')
```

### Option B: Glob for Newest Binary
Instead of using manifest, scan `build/` directory:

```python
binaries = glob('build/*-dirty.bin') + glob('build/*.bin')
newest = max(binaries, key=os.path.getmtime)
upload(newest)
```

### Option C: Track Build Output
Parse build output to extract versioned path:

```python
# Parse from build output line:
# "project-73e0af2-dirty.bin (versioned)"
versioned_binary = extract_versioned_path(build_output)
```

## Affected Commands

| Command | Impact | Workaround |
|---------|--------|------------|
| `esp32-p4 iterate` | ❌ Uploads wrong file | Manual two-step |
| `esp32-p4 flash-batch` | ❌ Uses stale manifest | Manual `--file` |
| `esp32-p4 upload` | ✅ Works if `--file` used | Always use flag |
